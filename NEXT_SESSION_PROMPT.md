- MVP(P0~P5): 인증/초대·태스크·오늘할일·가이드 팀원별 추적·첨부·웹푸시·SKILL.md 추출

작업 규칙(중요):
- HANDOFF.md의 "§4 개발 환경 제약"을 반드시 지켜: 서버/공용 코드에서 TS enum·파라미터 프로퍼티 금지, 서버/공용 상대 임포트는 `.ts` 확장자, 테스트는 `node --experimental-strip-types --test server/src/test/*.test.ts`(vitest 아님, PGlite+pgvector 사용).
- 코드 수정 후에는 `npm run check`(타입체크)와 `npm test`(통합 테스트 40개+) 둘 다 통과시켜. 테스트는 한 번에 다 돌리면 오래 걸리니 파일별로 나눠 실행해도 됨.
- 의존성을 추가하면 반드시 `npm install`로 package-lock.json을 갱신해(안 하면 Docker `npm ci` 실패).
- 마이그레이션은 `migrations/0000_init.sql`에 멱등 DDL(IF NOT EXISTS / ADD COLUMN IF NOT EXISTS)로 추가.
- 각 기능은 실데이터로 동작(mock UI 금지), 서버측 멤버십 필터·activity_log 기록·PATCH 화이트리스트 유지.
- AI 응답 자동 등록 금지 — 반드시 사람 검토 후 반영(§13).
- 각 단계마다 happy path + 권한 거부 테스트를 추가하고, 완료 시 HANDOFF.md에 기록해.
- 큰 작업이면 서브에이전트로 검증(보안 리뷰 + QA)까지 돌려서 완성도를 높여.
